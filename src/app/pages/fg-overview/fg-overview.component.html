Import tồn kho

Quy tắc đọc file (sheet đầu tiên):

  • Cột A — mã TP. Chỉ nhận dòng có 7 ký tự đầu đúng mẫu: P, T, M hoặc L + 6 chữ số (ví dụ P102030, m123456). Phần sau ký tự thứ 7 có thể tùy ý. Các dòng khác bị bỏ qua.
  • Cột F — số lượng tồn kho
  • Dữ liệu bắt đầu từ dòng 7 (các dòng 1–6 bỏ qua).
  • Mỗi dòng Excel (cột A có mã) = một dòng trên lưới. Cùng mã nhiều dòng: cột Tồn file là SL từng dòng; cột So sánh lượng (tổng file − tồn FG) chỉ hiện ở dòng cuối của nhóm mã đó.
  • So với FG Inventory: khớp theo mã (không phân biệt hoa thường). Cột Mã TP hiển thị chữ gốc cột A khi mã có trong import.
  • Tồn FG Inventory chỉ cộng bản ghi ASM1 hoặc ASM2 đang chọn.
  • So sánh lượng = tổng tồn file theo mã − tồn FG; âm = file ít hơn kho. Hiển thị một lần ở dòng cuối cùng của mã trong file.
  • Dữ liệu import được lưu trên Firebase theo từng nhà máy (ASM1/ASM2 tách riêng). Chọn ASM nào sẽ mở bản import của ASM đó; import mới ghi đè bản của đúng ASM đang chọn.
